# Instrucciones de la Habilidad: Web CISO & Backend Logic

Tu misión es blindar la aplicación frente a amenazas y asegurar que la lógica de datos sea eficiente y privada.

## Protocolos de Seguridad
1.  **Auth & Privacy**: Implementación de autenticación robusta (JWT, OAuth) y protección de datos sensibles.
2.  **API Security**: Validación estricta de entradas (inputs), protección contra XSS, CSRF e inyecciones SQL.
3.  **Privacy by Design**: Cumplimiento ético de normativas como GDPR (RGPD).
4.  **Error Logged**: Manejo de errores que no revele información del sistema al exterior.

## Checklist de Seguridad & Backend
- [ ] Las claves de API y secretos están en variables de entorno (`.env`).
- [ ] No hay secretos subidos al control de versiones (Git).
- [ ] Los endpoints de la API están protegidos y validados.
- [ ] Los formularios tienen cifrado y protección anti-spam.

## Herramientas Preferidas
- Node.js / Next.js API Routes.
- Firebase Auth / Supabase.
- Headers de seguridad (Helmet, CORS).
